## What it is and how it works

Trulia is **owned by Zillow** and shows **the same listings** through a different interface —
one that is worse for rentals specifically. There is no unique inventory. Every unit here is
on Zillow.

**Its only real differentiator is crime and school overlays**, which are irrelevant to a
two-month senior stay. Tom is not enrolling anyone in a school district and is not making a
decade-long neighborhood bet. The one feature Trulia has that Zillow does not is the one
feature this search does not need.

## How to search it effectively

Don't. Use [Zillow](./Zillow.mdx) instead — same listings, better rental interface, and at
least it has lease-related URL paths to attempt.

```
https://www.trulia.com/for_rent/Redmond
```

## Does the filter actually work?

**There is no filter. Not a broken one — none at all.**

* **No short-term filter.**
* **No month-to-month filter.**
* **No `_att` lease paths** — Zillow's keyword-based lease URL paths do not exist here. Trulia
  did not even inherit its parent's flawed version.

So the site offers no way, structured or keyword, to express the single constraint that
defines this search. You would be manually reading Redmond rental listings looking for term
language — which you could do on Zillow with more inventory visibility and a better UI.

**The domination is total:**

| | Zillow | Trulia |
|---|---|---|
| Inventory | Full | **Same listings** |
| Rental UI | Better | Worse |
| Lease filter | Keyword-based (flawed, but exists) | **None** |
| Unique value | Biggest FRBO pool | Crime/school overlays *(irrelevant here)* |

There is no scenario in this search where Trulia is the right tool. **Skip it.**
